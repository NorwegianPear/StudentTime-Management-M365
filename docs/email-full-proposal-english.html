Solution 2: Hybrid AD with Pass-through Authentication

How It Works

Sets up an on-premises Active Directory with Azure AD Connect and Pass-through Authentication. AD has built-in support for "Logon Hours" which is respected during authentication.

âœ… Advantages

  • Native Microsoft functionality
  • Best user experience
  • Students get clear error message
  • Granular control per user

âš ï¸ Disadvantages

  • Requires on-premises server infrastructure
  • Higher operational costs
  • More complex architecture
  • Not practical for cloud-only environment

Not recommended for your environment as you are 100% cloud-based.

Solution 3: Intune Device Control

How It Works

Uses Intune Compliance Policies and Device Configuration to restrict access at the device level, for example by locking devices outside school hours.

âœ… Advantages

  • Good user experience
  • Included in existing license
  • Can be combined with other policies

âš ï¸ Disadvantages

  • Only for Intune-managed devices
  • Does not work on personal devices
  • Students can log in via browser on other devices

Partial solution - Can be combined with Azure Automation for better coverage.

Solution 4: Conditional Access Policies

What It Can Do

Conditional Access in Entra ID can restrict access based on location, device, risk, etc. - but NOT based on time.

Important: Conditional Access currently does NOT support time restrictions. Microsoft has this on their backlog, but no ETA has been announced.

Can be used together with Azure Automation for extra security (e.g., require MFA, block risky sign-ins).
