📊 2. Monitoring Automation Jobs

How to View Job History

Azure Portal → Automation Accounts → StudentAccessAutomation → Jobs

Step-by-Step Instructions:

  1. 1 Open portal.azure.com and sign in
  2. 2 In the search bar at the top, type "Automation Accounts" and select it
    📷 Screenshot: Azure Portal search bar with "Automation Accounts"
  3. 3 Click on your automation account (e.g., StudentAccessAutomation)
  4. 4 In the left menu, under Process Automation, click Jobs
    📷 Screenshot: Automation Account left menu showing "Jobs" option
  5. 5 You'll see a list of recent jobs with their status

Understanding Job Status

Status Icon Meaning Action Required
Completed ✓ Job finished successfully None - working as expected
Running ↻ Job is currently executing Wait for completion (usually < 5 minutes)
Failed ✗ Job encountered an error Click to view error details, see Troubleshooting
Queued ⏳ Job waiting to start Will start automatically

Viewing Job Output

  1. Click on a specific job in the Jobs list
  2. Select the Output tab to see detailed logs
💡 What to look for in the output:
  • Total students processed
  • Number of accounts enabled/disabled
  • Any error messages

🔧 3. Manual Operations

⚠ When to use manual operations:
  • Testing the solution
  • Emergency enable/disable outside schedule
  • Verifying the automation works correctly

How to Run a Runbook Manually (Azure Portal)

Automation Account → Runbooks → [Select Runbook] → Start
  1. 1 In your Automation Account, click Runbooks in the left menu
    📷 Screenshot: Automation Account showing Runbooks menu item
  2. 2 Click on the runbook you want to run:
    • Enable-StudentAccess - to enable all students
    • Disable-StudentAccess - to disable all students
    • Get-StudentAccessStatus - to view current status
    📷 Screenshot: List of runbooks showing the three available runbooks
  3. 3 Click the Start button at the top
    📷 Screenshot: Runbook page with "Start" button highlighted
  4. 4 Confirm by clicking OK (no parameters needed)
  5. 5 Monitor the job in the Jobs view

Check Current Student Status

Run the Get-StudentAccessStatus runbook to see a report of all students and whether they are currently enabled or disabled.

📅 4. Managing Schedules

View Current Schedules

Automation Account → Schedules
  1. 1 In your Automation Account, click Schedules in the left menu
  2. 2 You'll see the configured schedules:
    Schedule Name Time Days Linked Runbook
    Enable-Students-Morning 07:55 AM Mon-Fri Enable-StudentAccess
    Disable-Students-Afternoon 04:05 PM Mon-Fri Disable-StudentAccess

Modify Schedule Times

  1. 1 Click on the schedule name you want to modify
  2. 2 Update the Start time
    📷 Screenshot: Schedule edit page with time field highlighted
  3. 3 Click Save
💡 Note: Changes take effect from the next scheduled run, not immediately.

Disable a Schedule Temporarily (e.g., for Holidays)

  1. Click on the schedule name
  2. Toggle Enabled to No
  3. Click Save
⚠ Holiday Reminder: Before school holidays, remember to either:
  • Disable the Enable-Students-Morning schedule, OR
  • Run Disable-StudentAccess manually on the last day
After the holiday, re-enable the schedule.

👥 5. Managing the Student Group

Find the Student Group

Entra Admin Center → Groups → All groups → [Search for your group]
  1. 1 Go to entra.microsoft.com
  2. 2 In the left menu, expand Groups and click All groups
    📷 Screenshot: Entra Admin Center left menu showing Groups
  3. 3 Search for your student group name in the search box
  4. 4 Click on the group to view details

Add a Student to the Group

  1. 1 Open the student group
  2. 2 Click Members in the left menu
  3. 3 Click + Add members
    📷 Screenshot: Group members page with "Add members" button
  4. 4 Search for the student by name or email
  5. 5 Select the student and click Select
✔ Result: The student will be included in the next automation run. They will be enabled/disabled according to the schedule.

Remove a Student from the Group

  1. Open the group → Members
  2. Find and select the student(s) to remove
  3. Click Remove
⚠ Note: Removed students will no longer be affected by the automation. Their account will remain in whatever state it was (enabled or disabled).

🔎 6. Troubleshooting

Problem: Job Failed with "Access Denied"

Possible Cause Solution
Client secret expired
  1. Go to Entra Admin Center → App registrations
  2. Find the StudentAccessAutomation app
  3. Check Certificates & secrets for expired secrets
  4. Create a new secret and update the Automation variable
API permissions not consented
  1. Go to Entra Admin Center → App registrations
  2. Open the app → API permissions
  3. Click "Grant admin consent"

Problem: Job Completes but "0 members found"

Possible Cause Solution
Wrong StudentGroupId
  1. Verify the group Object ID in Entra Admin Center
  2. Update the StudentGroupId variable in Automation Account → Variables
Group has no members Add students to the group (see Section 5)
Wrong group type Ensure it's a Security group, not an M365 Group

Problem: Schedule Not Running

Check Action
Schedule is disabled Go to Schedules → Click the schedule → Ensure "Enabled" is "Yes"
Schedule not linked Go to the Runbook → Schedules → Verify it's linked to the schedule
Wrong timezone Verify schedule timezone matches your location (W. Europe Standard Time for Norway)

How to View Error Details

  1. Go to Automation Account → Jobs
  2. Click on the failed job
  3. Click All Logs tab
  4. Filter by Error stream
📷 Screenshot: Job details page showing "All Logs" tab with error filter

Task How Purpose

How to Rotate the Client Secret (Annual)

⚠ Important: Client secrets expire. You must rotate them before expiration to avoid service interruption.
  1. 1 Create new secret:
    Entra Admin Center → App registrations → StudentAccessAutomation → Certificates & secrets
    • Click + New client secret
    • Set expiration (recommend 12 months)
    • Click Add
    • Copy the Value immediately! (it won't be shown again)
  2. 2 Update Automation variable:
    Azure Portal → Automation Account → Variables → ClientSecret
    • Click Edit
    • Paste the new secret value
    • Click Save
  3. 3 Test: Run Get-StudentAccessStatus manually to verify
  4. 4 Delete old secret: Return to App registrations and delete the expired secret

🚨 8. Emergency Procedures

Emergency: Enable All Students NOW

Scenario: School needs all students enabled outside normal hours
  1. Go to Automation Account → Runbooks
  2. Click Enable-StudentAccess
  3. Click Start
  4. Wait for completion (usually < 5 minutes)

Emergency: Disable All Students NOW

Scenario: Security incident requires immediate lockout
  1. Go to Automation Account → Runbooks
  2. Click Disable-StudentAccess
  3. Click Start
  4. Wait for completion

Emergency: Stop All Automation

Scenario: Need to completely stop the automation from running
  1. Go to Automation Account → Schedules
  2. Click Enable-Students-Morning → Set Enabled = No → Save
  3. Click Disable-Students-Afternoon → Set Enabled = No → Save

Result: No scheduled jobs will run until you re-enable the schedules.

Enable/Disable a Single Student

If you need to make an exception for one student:

Entra Admin Center → Users → [Search student] → Edit properties
  1. Find the student in Users
  2. Click Edit properties
  3. Under Account, toggle Account enabled
  4. Click Save
💡 Note: This change will be overwritten by the next automation run unless you also remove the student from the group.
