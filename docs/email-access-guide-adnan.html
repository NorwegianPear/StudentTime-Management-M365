🌐 Part 1 — Add biss.no as Custom Domain in Entra ID

⚠️ This step is optional — your M365 tenant works without it. However, adding biss.no as a verified domain lets you use @biss.no email addresses and UPNs instead of @bissstavanger.onmicrosoft.com. If you already use @biss.no addresses in M365, this is already done — skip to Part 2.

Steps to Add biss.no Custom Domain

1 Go to Microsoft Entra admin center
Open https://entra.microsoft.com and sign in with your Global Administrator account.
Navigate to: Settings → Domain names → Add custom domain
2 Enter the domain name
Type biss.no and click Add domain. Microsoft will give you a TXT record to add to your DNS.
3 Add DNS TXT record
Log in to your DNS provider (whoever manages biss.no). Add the TXT record Microsoft gives you. It looks like:
Type: TXT    Host: @ (or biss.no)    Value: MS=msXXXXXXXX DNS propagation can take 15–60 minutes.
4 Verify the domain
Back in Entra admin center, click Verify. Once verified, the domain shows as "Verified" with a green checkmark. You can optionally set it as the primary domain.

🔒 Part 3 — Assign Required Roles

Once you have invited/created my account (Part 2), assign these roles. These are the minimum permissions required to deploy the full solution.

3A — Microsoft Entra ID Roles (for M365 & App Registration setup)

Entra ID Role Why Needed Required For
🔴 Global Administrator Create App Registration + grant admin consent for Graph API permissions Script setup + Portal setup
Alternative if GA not possible:
Application Administrator + Privileged Role Administrator		 Application Admin creates app reg; Privileged Role Admin grants consent Script setup only

⚠️ Global Administrator is by far the simplest option. If your security policy requires a more restricted route, the "Application Administrator + Privileged Role Administrator" combination covers the script setup — but Global Admin is still needed to fully configure the portal's Managed Identity.

How to assign an Entra ID role:

  1. Go to entra.microsoft.com → Roles and administrators
  2. Search for Global Administrator → click it
  3. Click Add assignments
  4. Search for my account (guest or newly created user) → Select it → Click Add

3B — Azure Subscription Roles (for deploying Azure Automation + Portal)

⚠️ This requires an Azure subscription. If BISS does not yet have one, create a free Pay-As-You-Go subscription at azure.microsoft.com/free (includes $200 free credit). The automation solution itself costs approximately 0 NOK/month within the free tier limits.
Azure Role Scope Why Needed Required For
🟢 Contributor Subscription or Resource Group Create and configure all Azure resources (Automation Account, App Service, etc.) Script + Portal setup
🟢 User Access Administrator Resource Group Assign Managed Identity permissions to the App Service (portal only) Portal setup only
🔔 Tip: Assigning Owner at the Resource Group level covers both roles above and is the simplest approach.

How to assign an Azure role:

  1. Go to portal.azure.com
  2. Navigate to your Subscription (or to the specific Resource Group if you prefer a narrower scope)
  3. Click Access control (IAM) → Add → Add role assignment
  4. Role: Contributor → Next
  5. Members: Search for my account → Select → Review + assign
  6. Repeat for User Access Administrator if deploying the portal

📄 Part 4 — Scripts I Will Run in Your Environment

For full transparency, here are the scripts I will run and what each one does. I will also hand these over to you after setup so BISS can operate the solution independently.

Setup Scripts (run once during deployment — by Atea)

Script File What It Does Requires
New-AppRegistration.ps1 Creates the Entra ID App Registration with Microsoft Graph API permissions (User.ReadWrite.All, Group.Read.All, User.RevokeSessions.All) and generates a client secret Global Administrator in Entra ID
Deploy-StudentAccessAutomation.ps1 Creates the Azure Resource Group, Automation Account, uploads runbooks (Enable-StudentAccess.ps1 & Disable-StudentAccess.ps1), sets encrypted variables, and creates the daily schedules Azure Contributor on subscription/resource group
Deploy-Portal.ps1 (Optional — portal only) Deploys the web management portal to Azure App Service. Provisions infrastructure and uploads the Next.js portal build Azure Contributor + User Access Administrator
Grant-ManagedIdentityPermissions.ps1 (Optional — portal only) Grants Microsoft Graph permissions to the App Service's Managed Identity so the portal can call the Graph API without a client secret Global Administrator in Entra ID

Runbooks — Left in Your Azure Automation Account (daily operations)

Runbook File When It Runs What It Does
Enable-StudentAccess.ps1 Mon–Fri at 07:55 Sets accountEnabled = true for all students in the security group. Students can log in to M365.
Disable-StudentAccess.ps1 Mon–Fri at 16:05 Sets accountEnabled = false + revokes all active tokens. Students are immediately logged out and cannot log back in until 07:55 next school day.

Scripts Handed Over to BISS After Setup

Script File Purpose (for BISS IT to use)
Enable-StudentAccess.ps1 Manually re-enable access for a student or all students (e.g. for Saturday exam)
Disable-StudentAccess.ps1 Manually disable access for a student or all students (e.g. emergency lockdown)
Get-StudentAccessStatus.ps1 View current enabled/disabled status of all students in the group
Process-Suspensions.ps1 Handle individual student suspensions (keep a specific student disabled even during school hours)
Apply-SchedulePolicies.ps1 Apply custom schedule overrides (e.g. extended hours for exam periods, school holidays)
config.json (template) Configuration file — stores Tenant ID, Group ID, schedule times. Never contains secrets (those stay in Azure Automation encrypted variables).
