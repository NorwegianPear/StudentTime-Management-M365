🛠 Detailed Setup Guide

Step 1: Create App Registration in Entra ID

🔒 App Registration Setup (Entra ID Portal)
1. Go to entra.microsoft.com → App registrations → New registration
2. Name: StudentAccessAutomation
3. Supported account types: Single tenant
4. Click Register → Copy the Application (client) ID and Tenant ID
5. Go to Certificates & secrets → New client secret → Copy the secret value
6. Go to API permissions → Add permission → Microsoft Graph
Permission Type Purpose
User.ReadWrite.All Application Enable/disable user accounts
Group.Read.All Application Read student group members

⚠ Important: Click "Grant admin consent" after adding permissions!

Step 2: Verify/Create Student Security Group

👥 Security Group Configuration
1. Go to entra.microsoft.com → Groups → All groups
2. Search for existing student group OR create new: New group → Type: Security
3. Name: Students-M365 (or your preferred name)
4. Membership type: Assigned or Dynamic (based on user attributes)
5. Copy the Object ID (this is your StudentGroupId)
💡 Tip: If you already have a group with all students (from School Data Sync or manual), just copy its Object ID. No need to create a new one!

Step 3: Deploy Azure Automation

☁ Azure Automation Deployment
1. Go to portal.azure.com → Create a resource → Search "Automation"
2. Create Automation Account:
  • Name: student-access-automation
  • Region: Norway East (or closest)
  • Resource Group: Create new or use existing
3. After creation, go to Modules → Browse Gallery → Install:
  • Microsoft.Graph.Authentication
  • Microsoft.Graph.Users
  • Microsoft.Graph.Groups
4. Go to Variables → Add these (mark secrets as encrypted):
TenantId Your tenant GUID No
ClientId App registration ID No
ClientSecret App secret value 🔒 Yes
StudentGroupId Security group Object ID No
5. Go to Runbooks → Create runbook → Upload scripts from GitHub
6. Go to Schedules → Create two schedules:
  • Enable-Morning: 07:55 AM, Mon-Fri, Timezone: (UTC+01:00) Oslo
  • Disable-Afternoon: 04:05 PM, Mon-Fri, Timezone: (UTC+01:00) Oslo
7. Link schedules to runbooks: Runbook → Link to schedule
